# Compaction Hardening — Context

**Feature slug:** compaction-hardening
**Date:** 2026-07-23
**Exploring session:** complete (fresh-eyes reviewed; 4 P1 / 8 P2 / 5 P3 findings folded in)
**Scope:** Standard
**Domain types:** RUN (hooks), CALL (bee.mjs verbs), READ (doctrine docs + orientation output), ORGANIZE (a new log file)

## Feature Boundary

After a compaction, a bee session must land back on the work it was doing — with an
objective it can still read, orientation sized for the moment, a record that the
compaction happened, and an honest statement of which runtime enforces what. This
feature delivers those four things and stops there: it does not touch how any runtime
performs compaction, and it adds no provider-specific transport.

Prior art this builds on: `intent-anchor` (ia-1, v1.11.2) shipped the anchor store
(`skills/bee-hive/templates/lib/intent.mjs`), its PreCompact block and its
`SessionStart(compact|resume)` resume block. This feature closes that feature's own
recorded open gap and adds the three surfaces around it.
