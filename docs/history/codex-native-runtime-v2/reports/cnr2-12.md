# cnr2-12 — Plugin routes ship rendered per-runtime skill trees

**[DONE]** — high-risk, capped 2026-07-18.

## Outcome (one line)
Each plugin manifest now routes to a COMMITTED, renderer-only-generated per-runtime skill tree (`.claude-plugin/skills/` = `render(canonical, claude)`, `.codex-plugin/skills/` = `render(canonical, codex)`) instead of the raw canonical source — closing the raw-source-bleed blocker ahead of cnr2-10/11's tagging — with a drift-pin + runtime-clean regression net; zero markers today means both trees stay byte-identical to `skills/`.

## What changed
- **New generator** `scripts/render_plugin_skill_trees.mjs`: walks canonical `skills/bee-*`, gates the whole tree through cnr2-9's `validateSkillMarkers`, renders every file through cnr2-9's `renderSkillBytes(bytes, runtime)` for both `RENDER_RUNTIMES`, writes the committed trees, and stamps each with the D9 `.bee-render.json` provenance sidecar (`{schema:"bee-render/1", target_runtime}`) — the same sidecar `source-identity.mjs` already refuses as an onboarding source for any target.
- **Manifests repointed**: `.codex-plugin/plugin.json` `"skills"` changed from `"./skills/"` to `"./.codex-plugin/skills/"`; `.claude-plugin/plugin.json` gained an explicit `"skills": "./.claude-plugin/skills/"` (previously implicit, same canonical location).
- **Release integrity extended** (`scripts/release_manifest.mjs`): two new enumerated roots (`plugin_skill_claude_render`, `plugin_skill_codex_render`) alongside the unchanged canonical `plugin_skill` role; manifest grew from 146 to 354 records.
- **Distribution proof extended** (`skills/bee-hive/scripts/plugin_distribution.mjs`): the two new roles added to `PACKAGE_ROLES` so `proveInstalledPackage` counts the rendered trees as expected installed-package content, not "unexpected".
- **Regression net** (`skills/bee-hive/scripts/test_plugin_distribution.mjs`, +109 lines / 8 new checks): drift-pin (recomputes `render(canonical, runtime)` at test time and byte-compares against the committed tree), provenance-sidecar check, runtime-clean check (no unstripped full-line `bee:only`/`bee:end` marker survives — an exact-line match, not substring, so it doesn't false-positive on `onboard_bee.mjs`'s own comments documenting the marker grammar), release-inventory coverage check, and a manifest-routing check.
- **Doc fix** (`docs/02-architecture.md`): the now-false "both plugin manifests point at the same `skills/` directory" claim corrected to describe the per-runtime rendered-tree routing.

## Deviation
`scripts/test_installers_e2e.mjs`'s `stagedSource()` helper hardcodes its own package-role `Set`, independent of `plugin_distribution.mjs`'s `PACKAGE_ROLES`. Adding the two new roles there without updating this duplicate left its synthetic installed-package fixture missing the rendered trees, failing `proveInstalledPackage` with a false "missing" mismatch. Auto-fixed (deviation rule 1/3: bug in touched verify-chain path) — the two new roles added to that Set to match.

## Notes for downstream (cnr2-10 → cnr2-11)
- The plugin route is now the rendered tree, not raw canonical — cnr2-10/11's tagging work will show up in both plugin routes automatically once `scripts/render_plugin_skill_trees.mjs` is re-run to regenerate them (this cell's generator is idempotent and safe to re-run any time canonical `skills/` content changes).
- The runtime-clean test's exact-full-line marker match is deliberately NOT a substring search — `onboard_bee.mjs`'s own comments document the marker syntax inline (e.g. `//   <!-- bee:only claude -->   ...`) and a naive substring check false-positives on that legitimate prose. Any future runtime-clean check addition should keep the same exact-line discipline.
