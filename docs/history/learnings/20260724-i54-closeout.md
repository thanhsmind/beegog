# Learnings — i54-closeout (issue #54 full closeout)

## What Happened

Closed every remaining open item from the issue #54 cross-check against
v1.16.0 in one standard-lane feature: 10 cells, all capped green, wave-close
impacted run 57 suites PASS. The decisive move was live-probing codex 0.145.0
instead of arguing from any written surface: three corroborating probes (full
canary, native-override probe, model tool-schema self-inspection) proved
`spawn_agent` requires `task_name`, has no `agent_type`, and rejects model
override envelopes (`native_budget_only`). All three contradicting surfaces
(doc / helper / guard) were aligned on that observed truth. The canary also
caught an unplanned production bug: fresh repo-hooks installs shipped a write
guard that crashed at import (`tokenize-command.mjs` never vendored) and
therefore failed OPEN on the pre-Gate-3 write block.

## Root Cause (of the notable failures)

1. **Vendoring drift invisible to every existing test:** the crash sat inside
   the guard's fail-open try/catch; `node --check` passes on missing sibling
   imports; and every suite ran against the canonical source, where the import
   always resolves. Only an environment-faithful subprocess run from a
   freshly-onboarded fixture could see it.
2. **Recurring cell-authoring gap:** three frozen-judge hits in one feature
   (cells 3, 5, 8) had the same shape — a test file named in the cell's own
   action text but missing from `files[]`. The identical gap was already
   logged once in `transcript-recovery-3` and never mechanized.
3. **Red-first oracle coincidentally satisfiable:** cell 8's local reversion
   re-test stayed green because the dev machine's live codex equals the new
   pin — the oracle string was fed by live detection, not the constant under
   test. The worker root-caused it and substituted the external canary as the
   real proof, flagged via `deliberate_exceptions`.
4. **Regen obligation blind spot:** per-cell regen chains covered projections
   and the release manifest but not the impact registry; three new suites made
   it drift, caught only at wave close.
5. **Dangling PBI id:** `p-8612ec06` was cited by a cell and a bypass audit
   but never existed in `.bee/backlog.jsonl`.

## Recommendation

- **When vendoring/allowlisting ships files that import other files**, add an
  import-closure test (static parse of each shipped file's imports against the
  list) plus a fresh-install subprocess proof — canonical-source tests
  structurally cannot catch this class. (Mechanized: `test_hook_vendor_closure`.)
- **When a cell's action text names a test file, put it in `files[]` in the
  same edit.** The frozen-judge "justified" carve-out is a post-hoc patch, not
  a license — this has now recurred four times across two features; a
  mechanical authoring check is filed as friction.
- **When a red-first oracle can be satisfied by live-environment detection**,
  do not accept the local re-test as the red floor; require an
  environment-independent proof and flag the exception loudly.
- **When a cell adds/removes a test suite or changes suite imports**, its
  regen step must include `impact_registry.mjs --write` — never leave registry
  drift to wave close.
- **When any record cites a PBI id, the id must resolve to a real backlog
  row** before proceeding; mint-and-cite is an authoring defect.
- **When a schema/behavior spans doc + emitter + validator and they disagree**,
  live-probe the authoritative system during validating and align all surfaces
  on observation; accept model self-report only with independent corroboration.
- **When making a hardcoded command config-driven**, ship a byte-equivalent
  default, substitute per-token (never join/eval), and add an automated argv-
  capture regression test — manual stub verification alone is filed as debt.
- **When duplicated prose matrices must agree**, a semantic consistency test is
  the honest cheap guard; build a generator only when a third copy appears.
