# 2026-07-21 — A cleanup tool is a deletion tool: the scratch fix nearly deleted the store it was protecting

**Feature:** tree-hygiene (user report: bee writes many working files and forgets them) · **Tags:** [scratch, guards, testing, safety, deletion]

1. **The near-miss.** The first cut of `bee tmp sweep` resolved its two allowed roots (`.bee/tmp`, `.bee/spikes`) with `realpath` and trusted the *resolved* target as the delete authority. An independent judge reproduced, through the shipped CLI, that a symlinked `.bee/spikes` relocated that authority: the sweep deleted `.bee/cells/`, `.bee/decisions.jsonl`, and `docs/history/`. **Rule:** for anything that deletes, containment must be proved against the **literal** expected path — lstat each segment, refuse a symlinked root rather than following it — and re-proved immediately before each removal, in dry-run too.
2. **The test lesson is bigger than the bug.** The suite was 19 checks green while the single most safety-critical line could be deleted and everything still passed. A test that asserts "the happy path works" does not pin a safety property. **Rule:** for a safety-critical line, mutate it away and watch the suite fail — if it stays green, the property is unpinned no matter how many checks exist. (After rework: neutering symlink handling fails the suite; a second mutation shows an independent lstat layer still refuses — defense in depth, and the reason label is deliberately *not* over-pinned.)
3. **Gitignore answered the wrong half.** Every leaked file was already ignored, so `git status` stayed clean while **153 MB / 9,176 files** accumulated. Invisible is not absent. **Rule:** cleanup needs an owner and a moment (feature close), not just an ignore rule.
4. **One home beats three good rules.** Three skills each forbade scratch in their own way and each named a different throwaway location; scratch still landed in tracked `.bee/bin/` because nothing enforced any of them. Collapsed into doctrine R17 plus a write-guard that denies scratch-shaped writes outside the scratch homes — with the *allow* side (a report or a cell whose name looks scratch-shaped) as the load-bearing test cases, since a false deny is worse than the garbage.
5. **Ask what the user actually said.** The first shaping targeted `.bak` backups; the user corrected it — the complaint was bee's own working documents, forgotten. The measured 153 MB only surfaced after re-reading the request. **Rule:** when a report and your inventory disagree about what the problem is, the report wins and the inventory gets re-run.
