# Learnings — pre-162-fixes (2026-07-19)

Feature: the user's 1.6.1 review "trước 1.6.2" list — doctor host-topology
(P1), worktree-merge staged transaction (P1), base-ref rev-parse (P3).

## 1. Dogfood topology hides host bugs — fixtures must model the HOST, not the dev repo
Doctor checked `<repo>/hooks/` because bee's own repo has that dir; every
test fixture copied bee's topology, so the bug was structurally invisible.
When a tool inspects "the repo it runs in", at least one fixture must be a
minimal HOST repo (only what onboarding actually installs), not a clone of
the dev repo's layout. Same class as the cph-1 lesson (cleanup path unread) —
the failure mode lives where the environment differs from the dev machine.

## 2. Stage-then-verify beats commit-then-apologize for any mutating transaction
The merge redesign (--no-commit, verify on the staged tree, abort with a
three-part main-untouched proof, commit only on green) removed a whole class
of "main is wounded" states without losing the semantic-conflict alarm. The
proof must be explicit (HEAD sha + MERGE_HEAD absent + tracked-clean):
porcelain-empty alone lies when verify creates untracked files. Abort belongs
in a `finally`.

## 3. Test suites accumulate hidden fixture chains — auditing them is part of changing a contract
The red-verify e2e left a flag flipped and a later case silently depended on
it. Changing the contract (abort instead of persist) collapsed the premise.
When a behavior contract changes, grep the suite for every reuse of the
fixture repo and make dependent cases self-establishing.

## 4. One typed refusal beats two when the underlying tool can't distinguish causes
rev-parse can't tell bad syntax from a missing ref; keeping two error codes
would have meant lying in one of them. Collapse to the honest single code and
retire the old one everywhere (tests included) in the same change.
