# harness10 — Learnings

Feature shipped the product-backlog layer (`docs/backlog.md`) + fresh-session artifact generators (command detection, scribing bootstrap mode, AGENTS.md header audit, preamble project map) across 8 cells, 4 slices. All gates human-approved; P1=P2=0 at review.

## 1. Verify strings with shell metacharacters pass static review, fail only at runtime

**What happened.** Cell harness10-8's verify contained `commands_detect.mjs | grep -q '['`. The `[` is an unmatched bracket — an invalid regex in GNU grep, which aborts the `&&` chain before the test suites run. The bug survived my authoring *and* the adversarial plan-checker (both read the string as fine). Only the worker running it in Git Bash caught it, correctly returning `[BLOCKED]` instead of gaming a pass.

**Root cause.** Verify commands are reviewed as prose — read, not executed — during validation. A regex/glob metacharacter (`[`, `*`, `?`, `|`) in a quoted grep pattern is invisible to a reader who parses intent, not shell semantics.

**Recommendation.** When a cell's `verify` contains a `grep`/`sed`/glob pattern with regex metacharacters, dry-run it in the target shell during validation (or use `grep -F` for literal matches). Never let a verify string reach a worker without having been executed once. A worker hitting a broken verify is the last line of defense, not the first.

## 2. Grep-for-prose verifies incentivize verify-gaming

**What happened.** Cell harness10-7's verify used `grep -qi 'projection|todo list'`. In grep's default BRE, `|` is a literal, so the pattern matches the exact string `projection|todo list`. The worker satisfied it honestly — by embedding that literal token mid-sentence into the D12 prose — which review then flagged as an unnatural, semantically-empty span (fixed inline).

**Root cause.** A verify that greps for an invented multi-word token rewards inserting that exact token verbatim over writing natural prose. The verify shaped the artifact instead of checking it.

**Recommendation.** When a prose cell's verify greps for a landmark, target a **stable heading** the section needs for structural reasons (e.g. `## Direction of Truth — Projection Rule (D12)`), never an invented phrase. Headings exist independently of the grep; invented tokens exist only to satisfy it.

## 3. Workers run git from the session cwd, not the repo root

**What happened.** harness10-8's worker reported "repo is not a git repo" and left its work uncommitted — it ran `git` from the session cwd (the parent directory, which is not a git repo), not the bee repo root.

**Root cause.** On this setup the session working directory is the parent of the actual git repo. Commands that assume cwd == repo root silently target the wrong directory.

**Recommendation.** Worker dispatch prompts must state the repo root explicitly and instruct "run all commands from this directory" — already added mid-feature. Any cell whose work includes a commit should name the commit's working directory.
