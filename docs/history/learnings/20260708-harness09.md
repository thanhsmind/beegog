# Learnings — harness09 (learn-harness-engineering adoption, slices 1–5)

## Learning: gate approvals must be recorded the moment they are given

**What happened:** Gate 2 was approved in chat but `approved_gates.shape` stayed `false`
until claim time — the orchestrator updated `phase` on the transition and forgot the gate
field. Nothing broke (claim only checks `execution`), but state and reality disagreed for
several steps, and `bee_status` would have misreported the gate to a resuming session.

**Root cause:** phase transition and gate recording are two writes with no mechanical
coupling; skills' state-update snippets show the phase but the gate write is prose.

**Recommendation:** When presenting any gate, write the approval into
`.bee/state.json.approved_gates` in the same turn the user answers — before the phase
transition. Candidate mechanization (filed to backlog): `bee_status` staleness warning
when phase has advanced past a gate that is still `false` (the post-review variant of
this check already exists for Gate 4).

## Learning: the enforcement layer teaches when messages carry the fix

**What happened:** `capCell` refused a behavior_change cap without `--evidence-file`; the
message named the exact flag and the work continued in one correction, no retry loop.
Separately, two `claimCell` messages were found fix-less during the slice-5 audit and
strengthened.

**Root cause:** message quality was implicit style, not a stated contract, so it drifted
per author.

**Recommendation:** When writing any refusal path, include the next command/action in the
same string — the contract now lives in 07-contracts (ERROR/WHY/FIX) with shape tests for
the three highest-traffic refusals.

## Learning: Windows Git Bash `/tmp` is invisible to node

**What happened:** `node -e "require('/tmp/x.json')"` failed — Git Bash maps `/tmp` to an
MSYS path node cannot resolve; shell redirection into `/tmp` works, node path resolution
does not.

**Root cause:** two path universes in one command line (POSIX shell + Windows node).

**Recommendation:** When a node process must read a file created via shell redirection on
Windows, pipe it through stdin (`cat file | node -e ...`) or use a Windows-style absolute
path; never hand a `/tmp/...` string to node APIs.
