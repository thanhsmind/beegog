# Learnings — advisor (worker-level on-failure consult)

## What Happened

One day after advisor mode was removed (de967733, "ONE cost pattern"), the owner
requested a different advisor shape: a stuck down-tier worker consults a
configured stronger model from inside its own turn, on failure only. Exploring
resolved the collision by **amending, not reversing** the fresh decision (owner
pre-approved the amendment at Q1), which defused the reversal-churn smell before
planning started. Shipped as 3 cells (resolver + executing prose + swarming
prose), all goal-checked green, specs synced same session.

## Root Cause / Key Findings

1. **A plausible wrong baseline nearly locked.** D2's degenerate check was first
   written against the *session* model; fresh-eyes caught that under fan-out the
   session is already the strong party — the honest baseline is the *consulting
   worker's own model*. A framing note was needed to stop the anecdote's "cheap
   main loop" from being read as the session.
2. **The obvious reuse path was a trap.** `resolveTier('advisor')` would coerce a
   null slot into the generation fallback — exactly what D2 forbids. A sibling
   resolver (`resolveAdvisor`) + a local normalize list left the tested vocabulary
   (`CONFIGURABLE_SLOTS`, decision 0015) byte-untouched.
3. **A proven fact still failed cold-pickup review when the cell did not cite its
   own proof.** adv-3's transport was fully proven by probes, yet drew 2 CRITICALs
   because the cell text neither referenced the validation report nor could — the
   report did not exist yet. Fix was referential, not investigative.
4. **Attribution beat machinery.** A2 (consults recorded) was nearly scope-reduced
   to a markdown section; probe C showed worker-originated Agent dispatches land in
   `.bee/logs/dispatch.jsonl` via the existing hook, so a description-prefix
   convention (`advisor-consult <cell-id>: <advisor>`) delivered machine-readable
   per-cell records with zero new storage.
5. **Provider error mid-cap, rescued by resume.** Worker bob (adv-3) died to an API
   error after verify passed but before cap; a resume message to the same agent
   finished cap/release/report with the worktree intact. Nothing in the cell trace
   records the transport fault — absorbed faults are invisible to trend detection
   (filed as friction).
6. **Shared-suite verify conflates siblings.** adv-2's verify ran the full suite
   while adv-1 was claimed-but-uncapped → transient 219/222 red that was NOT
   adv-2's defect. The worker correctly refused to touch the sibling's files and
   re-ran after the cap.
7. **Phase-enum drift recurred twice in one session** (`exploring-complete`,
   `validated` — both prescribed by skill prose, both rejected by the CLI enum).
   Skill prose was never round-tripped against the live enum.

## Recommendations

- When a new config slot has different semantics from an existing family (no
  fallback, no default), add a **sibling resolver + local normalize list**; never
  special-case the tested resolver (finding 2, pattern reusable).
- When writing a cell whose action states a fact proven elsewhere, put the proving
  artifact in `read_first` and make the action cite it — a proof the cell cannot
  reach is an unproven claim at cold pickup (finding 3).
- Before diagnosing a shared-suite red as a real defect, check `.bee/cells/` for
  claimed-but-uncapped siblings; re-run after they cap (finding 6).
- Cell `deps` must be justified by concrete file/output overlap, not authoring
  order (validation near-miss: adv-2's artificial dep on adv-1).
- For "record X per unit" requirements, first check whether an existing audit
  surface already sees the event — a naming convention on an already-logged field
  beats new storage (finding 4).
- When a provider/transport error is absorbed by a resume, still log one line
  (worker, cell, error class, resume method) — filed as friction for a capture
  point (finding 5).
- When skill prose prescribes a state value, verify it against the live enum in
  the same edit (finding 7; 2 P3 frictions filed, consolidation candidate).
