# Advisor digest — worktree-session-routing (high-risk lane)

Advisor: fable (ceiling/advisor tier, AO3/AO13 Gate 3 precondition). Consulted 2026-07-18.

**1. Verdict: GO-WITH-CONDITIONS.** The shape is sound: it builds only the named remainder of a shipped feature, reuses `bootstrapWorktreeStore`/grant plumbing verbatim, keeps every mutation behind a typed zero-mutation refusal, and correctly treats a red post-merge verify as the *report*, never a rollback. Conditions are the specific git-semantics traps below — the plan's Risks section names the umbrella ("typed zero-mutation refusals in e2e fixtures") but never the individual blades.

**2–3. Sharpest uncovered risks + required mitigation (each must be a test, not a claim):**

- **R1 — "dirty worktree tree" is undefined and will either block every merge or hide real dirt (D8).** A bootstrapped worktree always has gitignored cache/runtime `.bee/` files; if the dirtiness probe runs with `--ignored` it flags *every* worktree and merge never fires. *Mitigation:* define dirty strictly as `git -C <root> status --porcelain` (no `--ignored`) over tracked paths only; test that merge succeeds against a worktree that has a fully-bootstrapped store plus committed cell work.
- **R2 — slug / baseRef injection into path, branch ref, and argv.** `../<basename>--wt--<slug>` + `-b wt/<slug>` let a slug like `../../x`, a leading `-`, or shell metacharacters escape the sibling dir, forge a flag, or produce an invalid git ref. *Mitigation:* allowlist `^[a-z0-9][a-z0-9-]*$` for slug, validate baseRef via `git check-ref-format`, run every git call through `execFileSync` with an argv array (never a shell string) and a `--` terminator before user-derived values. Test the rejects.
- **R3 — partial state when `git worktree add` succeeds but `writeGrant`/`bootstrap` throws.** Order is irreversible-git-first, so a later throw leaves an orphaned linked worktree with no grant. *Mitigation:* on any post-add failure, best-effort `git worktree remove --force` the just-added tree and emit a typed error naming the path; if rollback itself fails, tell the user it is adoptable via `register`. Test the injected-failure path. Also: derive the grant `id` by reading it back from `.git/worktrees/<id>` after add — do **not** assume `id === slug` (git dedupes/renames the dir name).
- **R4 — `--cleanup` ordering: branch delete and worktree remove.** `git branch -d` (correct — never `-D`) refuses an unmerged branch — fine post `--no-ff`, but it also refuses while the branch is still checked out in the worktree, so **remove the worktree before deleting the branch**. `git worktree remove` refuses on modified/untracked files. *Mitigation:* order remove→branch-delete; decide and document the `remove` vs `remove --force` policy (the log tier already union-merged into main, so `--force` is safe for the disposable store) and test cleanup against a *used* worktree, not an empty one. Keep `-d` so an unexpectedly-unmerged branch is preserved, not destroyed.
- **R5 — merge run from the wrong cwd / removing your own worktree.** *Mitigation:* refuse `merge` unless invoked from the ordinary main checkout, and refuse when `--id` resolves to the cwd's own worktree. Rely on git's atomic failure (not just the TOCTOU pre-check) as the real guard for path/branch/grant collision in `new` — catch and type that failure too.

**4. Ambiguous decisions to pin before Gate 3:**

- **D8 "refuse on dirty ... worktree tree"** — quoted phrase is undefined; record R1's resolution (tracked-only porcelain, ignored tiers excluded), else merge is either always-blocked or false-clean.
- **D8 "offer cleanup (`--cleanup` flag)"** — "offer" vs a flag conflicts under total-autopilot where no human accepts an offer. Resolve: `--cleanup` performs cleanup unconditionally on green verify; without the flag, report the suggestion only, never prompt.
- **D9 "a mid-swarm default pipeline"** — no concrete detection signal for a *starting* session. Resolve to an observable: a live cross-session heartbeat plus a non-idle phase in the shared store; otherwise the prose is unfalsifiable. Low stakes (prose, no hook) but pin it so the render is testable.
