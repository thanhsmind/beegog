# docs/knowledge/ — Bee OKF Bundle

This is bee's own OKF v0.1 bundle (D17): the single `docs/knowledge/` directory holding curated
current truth, checked by `bee knowledge check` against the Bee OKF Profile
(`docs/specs/okf-profile.md`). It is **hand-seeded** for now. From **S3** on, this file is
generated byte-identically by `bee knowledge index` (D21) — every section below is replaced by the
generated index over the concepts that exist at that time; this hand-authored version exists only
to give the bundle a legal root before the generator ships.

Root frontmatter carries **only** `okf_version` (OKF §9; D4) — no other key is legal here.

## areas/

`bee.area` concepts: current truth about a subsystem, one or more per `areas/<slug>/` (authority is
per-subject via `bee.authoritative_for`, not per-directory — D31). Empty until a legacy
`docs/specs/*.md` area migrates in; `advisor-protocol.md` is F1's proof area (D29).

## features/

`bee.feature` concepts. Empty in this feature.

## work/

`bee.work-item` / `bee.plan` / `bee.delivery` concepts, one directory per unit of work
(`work/<id>/`). Empty until this feature's own work item lands at `work/okf-foundation/` (D34).

## decisions/

`bee.decision` concepts. Empty until decisions migrate into the bundle; `.bee/decisions.jsonl` and
`docs/decisions/` remain the authoritative decision record until then (`docs/decisions/index.md`
keeps its own path permanently — D11).

## patterns/

`bee.pattern` concepts, including pitfalls (`bee.polarity: pitfall` — D18 folds pitfall into
pattern rather than carrying a tenth type). Empty until `docs/history/learnings/critical-patterns.md`
migrates here as a generated index over `bee.critical: true` concepts (D21, D34).

## runbooks/

`bee.runbook` concepts. Empty in this feature.
